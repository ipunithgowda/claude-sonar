# claude-code-jarvis

> Turn Claude Code into JARVIS. Contextual voice status line after every turn — generated fresh by an LLM, spoken in a British butler tone, under 300ms to first word.

This is the exact setup I built to give my Claude Code assistant — callsign **AXE** (*Automated X-platform Engine*) — a JARVIS-style voice. Every time Claude finishes a turn, a Stop hook reads the message, asks a fast LLM to summarize it into a 5–12 word spoken status line, and speaks it with a natural British voice. No canned phrases. No robot "task complete." Actual context, every time.

**What it sounds like** — real lines pulled from my `voice.log` while I was working:

```
[00:18:06]  Shipment of all three projects has been successfully processed, sir.
[00:30:06]  Validation of Romeo, Oscar and Quebec systems proceeding in parallel, sir.
[00:37:19]  All three hero demos have passed validation, sir.
[04:11:33]  Axe, sir, all critical issues resolved, zero remaining.
[04:17:34]  Server shutdown initiated, sir, all systems offline now.
[04:21:33]  Shipping confirmation received, sir.
```

Those aren't scripted. Each one was generated by an LLM from the tail of the previous Claude turn. That's the magic — the voice *understands what just happened* and says it back to you like Paul Bettany.

## The Journey — Three Failed Designs Before This Worked

I didn't land on this stack immediately. I burned a day on two dead ends before the third attempt clicked. The story matters because if you're building your own, you'll be tempted by the same bad ideas.

### Attempt 1 — Literal text-to-speech of the whole reply

**The plan:** pipe Claude's full response to `edge-tts` and play it.

**Why it failed:** Claude's responses are *engineering text*. They contain code blocks, JSON payloads, file paths, shell commands, and markdown syntax. Hearing `"run tilde slash dot config slash claude slash hooks slash axe hyphen voice dot sh"` spoken aloud is not a JARVIS experience. It's an accessibility tool for a blind developer — which is a valid use case, but not what I wanted. I wanted **ambient status**, not a screen reader.

Rejection quote from my own memory: *"read test commands and raw JSON aloud — AI slop."*

### Attempt 2 — Six-bucket regex classifier with cached MP3s

**The plan:** regex-match the last message into one of six buckets — `done`, `clarify`, `permission`, `error`, `working`, `ready` — and play a pre-generated MP3 for each.

**Why it failed:** the classifier was wrong on most turns. A message like *"I'm going to run the migration and then check the schema"* could match `working`, `clarify`, *or* `permission` depending on whether I wanted to confirm first. And even when it picked the right bucket, it said `"Ready, sir."` — with **no context**. The voice was present, but it knew nothing. Dead air dressed up as intelligence.

This is the trap: classification feels cheap and fast, but the moment you have more than ~4 buckets, you're building a mini-NLU system from regex. You'll lose.

### Attempt 3 — Groq LLM summarizer (the one that works)

**The plan:** send the last assistant turn to a **fast, cheap** LLM (Groq's `llama-3.1-8b-instant`, ~300ms round-trip, free tier 30 req/min) with a strict JARVIS system prompt. Take the output, pipe it through `edge-tts`, play with `afplay`. Do it all async in a subshell so the hook returns instantly and the voice catches up a half-second later.

**Why it works:**
1. **Real comprehension.** The LLM actually reads the message. It knows *what* completed, *what* failed, *what* is being asked. The output is specific, not generic.
2. **Fast enough.** Groq's 8B model on their custom silicon returns in ~300ms. By the time the Glass.aiff chime plays, the voice is ready.
3. **Free enough.** 30 requests/minute on the free tier is way more than a human generates in real coding. I've never hit the limit.
4. **Degrades gracefully.** If Groq times out or the key is missing, the hook silently falls back to a cached MP3 or just... stays quiet. Voice is a cherry on top, not a load-bearing feature.

The lesson: **don't classify, summarize.** A small LLM is the right hammer when the task is "turn arbitrary text into a short contextual line." Regex isn't. Classification isn't.

## The Final Stack

```
┌─────────────────────┐    ┌─────────────────┐    ┌────────────────┐    ┌──────────┐
│  Claude Code        │    │  Stop hook      │    │  Groq          │    │ edge-tts │
│  finishes turn      │───▶│  axe-voice.sh   │───▶│  llama-3.1-8b  │───▶│  Ryan    │───▶ afplay
│  (writes transcript)│    │  tail -r JSONL  │    │  (~300ms)      │    │  Neural  │
└─────────────────────┘    └─────────────────┘    └────────────────┘    └──────────┘
                                   │
                                   │ fork async, disown
                                   ▼
                           hook returns in <10ms
```

| Component | Choice | Why |
|-----------|--------|-----|
| **Summarizer** | Groq `llama-3.1-8b-instant` | ~300ms round-trip, free tier, plenty smart for 12-word summaries |
| **Voice engine** | `edge-tts` (Python pip package) | No local model, streams from Azure free tier, ~1MB install |
| **Voice** | `en-GB-RyanNeural` | British male, composed — A/B/C tested against Thomas and Guy |
| **Player** | `afplay` | Built into macOS, zero deps |
| **Chime** | `Glass.aiff` | Runs in parallel with the voice for the "ding → speech" effect |
| **Wiring** | Claude Code `Stop` hook | Fires after every assistant turn |
| **Secrets** | `~/.claude/hooks/.env` (chmod 600) | Never in `settings.json`, never in git |
| **Fallback** | 6 pre-generated MP3s in `voice-cache/` | Only used if Groq errors mid-call |

## Install (5 steps)

Prereqs: macOS, Python 3.10+, `jq`, a free [Groq API key](https://console.groq.com/keys).

### 1. Clone to `~/.claude`

```bash
git clone https://github.com/filmy-munky/claude-code-jarvis.git ~/claude-code-jarvis
mkdir -p ~/.claude/hooks
cp ~/claude-code-jarvis/hooks/axe-voice.sh ~/.claude/hooks/axe-voice.sh
chmod +x ~/.claude/hooks/axe-voice.sh
```

### 2. Install `edge-tts`

```bash
python3 -m pip install edge-tts
```

`edge-tts` streams from Microsoft's Azure free tier. No API key needed. ~1MB install.

### 3. Drop in your Groq key

```bash
cp ~/claude-code-jarvis/hooks/.env.example ~/.claude/hooks/.env
chmod 600 ~/.claude/hooks/.env
# then edit ~/.claude/hooks/.env and paste your GROQ_API_KEY
```

Get a free key at [console.groq.com/keys](https://console.groq.com/keys). Free tier is 30 req/min — more than enough.

### 4. Wire the Stop hook into Claude Code

Merge the snippet from `settings.json.example` into `~/.claude/settings.json`:

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          { "type": "command", "command": "afplay /System/Library/Sounds/Glass.aiff &", "timeout": 5 },
          { "type": "command", "command": "/Users/YOU/.claude/hooks/axe-voice.sh", "timeout": 10 }
        ]
      }
    ]
  }
}
```

Replace `/Users/YOU/` with your actual home.

### 5. (Optional) Adopt the JARVIS persona in `CLAUDE.md`

Copy `CLAUDE.md.example` to wherever your Claude Code project-level or user-level instructions live. This is what teaches the *text* to match the *voice* — the model writes the kind of message the voice should speak.

That's it. Open a new Claude Code session and say hi. After the first reply, you should hear a Glass chime followed by a spoken one-liner.

## Customizing

**Different voice?** Swap `EDGE_VOICE` in `axe-voice.sh`. Try `en-GB-ThomasNeural`, `en-US-GuyNeural`, or `en-GB-LibbyNeural`. Full list: `python3 -m edge_tts --list-voices`.

**Smarter summaries?** Swap `GROQ_MODEL` to `llama-3.3-70b-versatile`. Slower (~800ms) but noticeably more nuanced.

**Different callsign?** Search/replace `AXE` → `your-name` in `axe-voice.sh` *and* `CLAUDE.md.example`. Keep it one word — multi-word or punctuated names (`A.X.E.`, `J.A.R.V.I.S.`) make TTS engines spell it out letter by letter. Don't ask how I learned that.

**Different address?** Replace `"sir"` in the system prompt with whatever you prefer. `"boss"`, `"chief"`, your first name — all work.

## Troubleshooting

**Silent after install.** Check `~/.claude/hooks/voice.log` — every invocation logs there. Common causes: missing `GROQ_API_KEY`, missing `jq`, missing `edge-tts`, or `PYTHON_BIN` in the script pointing at a Python where `edge-tts` isn't installed.

**Voice says "A dot X dot E dot".** Your system prompt or persona file has the name as `A.X.E.` somewhere. Replace with `AXE`.

**Voice is generic ("Ready, sir")** on every turn. You're hitting the fallback cache — the Groq call is failing. Check `voice.log` for curl errors or HTTP status. Most common: expired key or wrong env var name.

**Voice overlaps previous voice.** The script already runs `pkill -x afplay` before playing. If it's still overlapping, something else is holding the audio device. Check `lsof | grep afplay`.

## Why "AXE"?

**A**utomated **X**-platform **E**ngine. It's the callsign I gave my Claude Code setup. Pairs with the JARVIS persona from Iron Man — respectful, sharp, composed, addresses you as *sir*. Pronounced like the tool (rhymes with "tax"), not spelled out.

You can pick your own. But "AXE" is short, spells cleanly in a TTS engine, and — critically — doesn't collide with anything else in your shell history.
